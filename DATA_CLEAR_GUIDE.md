# 数据清空指南

## 概述

系统已移除所有硬编码的测试数据，现在支持完全自定义的数据测试。

## 已移除的硬编码数据

以下模块的硬编码数据已全部移除：

1. **商品管理**
   - 商品数据
   - 色号数据
   - 缸号数据

2. **往来单位**
   - 客户数据
   - 供应商数据

3. **进货管理**
   - 进货单数据

4. **销售管理**
   - 销售单数据

5. **染色加工**
   - 加工单数据

6. **账款管理**
   - 应收账款数据
   - 应付账款数据
   - 收付款记录

7. **库存管理**
   - 库存调整单数据
   - 盘点单数据

8. **打印管理**
   - 打印模板数据

9. **系统设置**
   - 门店信息（已清空，需重新配置）
   - 员工列表
   - 角色管理
   - 自定义查询
   - 库存预警设置（已禁用）

## 保留的数据

以下数据**保留**硬编码：

- **使用教程**：`src/pages/settings/TutorialManagement.tsx` 中的教程数据

## 如何清空数据

### 方法1：通过系统设置页面

1. 进入 **系统设置** → **清空数据**
2. 选择以下操作之一：
   - **清空所有数据**：清空所有业务数据
   - **按模块清空**：选择特定模块进行清空

### 方法2：通过浏览器控制台

在浏览器控制台（F12）中执行：

```javascript
// 清空所有数据
localStorage.clear()

// 或按模块清空
localStorage.removeItem('products')
localStorage.removeItem('customers')
// ... 等等
```

## 清空后的状态

清空数据后，系统将处于以下状态：

- ✅ 所有业务数据为空（商品、订单、账款等）
- ✅ 系统设置恢复默认值（门店信息为空，预警设置禁用）
- ✅ 教程数据保留
- ✅ 系统功能正常，可以开始添加新数据

## 开始测试

清空数据后，建议按以下顺序添加测试数据：

1. **系统设置**
   - 配置门店信息
   - 添加员工
   - 配置角色和权限

2. **商品管理**
   - 创建商品
   - 添加色号
   - 添加缸号

3. **往来单位**
   - 添加客户
   - 添加供应商

4. **业务单据**
   - 创建进货单
   - 创建销售单
   - 创建其他业务单据

## 注意事项

⚠️ **警告**：清空数据操作**不可逆**！请确保已备份重要数据。

## 技术说明

- 所有数据存储在浏览器的 `localStorage` 中
- 清空数据会删除所有业务数据，但不会影响系统功能
- 教程数据存储在代码中，不会被清空
